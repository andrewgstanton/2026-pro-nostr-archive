**Every Version Matters**

Andrew G. Stanton – July 6, 2026

Most software only remembers the latest version of your work.

Edit a note? The previous version disappears.

Update an article? The old one is gone.

Change your profile? Only the newest metadata survives.

That works fine if your application is only concerned with the present.

Continuum is beginning to take a different approach.

Every time I create or edit one of my own replaceable Nostr events, Continuum now preserves a local copy before moving forward. Those historical versions become part of my local archive—not just the latest state.

[Screenshot: Note showing the collapsed "Versions (3)" section and expanded list of historical revisions.]

This isn't about undo.

It's about memory.

Each revision remains a signed Nostr event with its own event id and timestamp. The current version is still the latest replaceable event on the network, but locally I can see how that note evolved over time.

Eventually this will expand beyond notes:

Articles
Profile metadata
Side-by-side comparisons
Restore an earlier revision by creating a new version

Notice that last point.

"Restoring" doesn't rewrite history. It creates a brand new event whose content comes from an earlier revision. The history remains intact.

That's a subtle but important distinction.

Git works the same way.

History isn't erased.

It's extended.

Most social clients focus on the latest event.

Continuum is beginning to focus on something different:

the history of your thinking.
