Offline-Safe Scheduled Publishing for Notes

Today I confirmed an important behavior in Continuum’s notes workflow: scheduled publishing is now offline-safe.

This means authors can continue working normally even when the network is unavailable.

The system now behaves as follows.

When the network is offline:

• Notes can still be written  
• Notes can still be signed locally  
• Notes can still be scheduled for future publication  
• Relay operations such as publishing or refreshing data are disabled  

If a scheduled publish time arrives while the network is offline, the event does not fail and does not attempt to publish. Instead, the note simply remains in the **scheduled** state.

Once network connectivity returns and Continuum is restarted, the scheduler processes the queue and publishes the note normally.

In other words, the author’s intent is preserved even when the network is unavailable.

The flow now looks like this:

Write → Sign → Schedule → (offline period) → Network returns → Scheduler publishes

This is exactly the behavior you want from a local-first publishing system. The author’s work and intent remain durable locally, and relay publishing becomes an opportunistic step that occurs when network access is available.

Importantly, offline periods do not produce misleading states such as “failed” or “missed.” The scheduled note simply waits until the system is able to execute the publish.

When the network is offline, the interface reflects this clearly:

• Publishing actions are disabled  
• Delete operations are disabled  
• The UI displays a banner indicating relay services are unavailable  

But the writing workflow continues uninterrupted.

Authors can draft, sign, edit, and schedule notes locally. Relay interaction only occurs when it is actually possible.

This small behavior change represents something deeper: Continuum continues to move toward a **local-first model of authorship**, where the author controls the workflow locally and the network becomes a distribution layer rather than a dependency.

In practice, Continuum now supports what I would call **offline-safe scheduled publishing for notes**.

A small step technically — but an important one architecturally.
