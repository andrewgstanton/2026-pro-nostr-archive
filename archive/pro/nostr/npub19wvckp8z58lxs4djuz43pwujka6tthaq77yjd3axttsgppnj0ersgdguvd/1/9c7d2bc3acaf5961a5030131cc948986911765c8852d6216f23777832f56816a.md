Today I chose to back Continuum's Media Library with Cloudflare R2 instead of Blossom.

That isn't a criticism of Blossom. 

I still think Blossom is an excellent protocol for Nostr media and expect to support it in Continuum as well.

For article assets, though, I wanted a few things:

• A stable URL under my own domain (media.mycontinuum.xyz)

• Immediate uploads without waiting for a Git deployment

• Storage that isn't tied to a particular Nostr service

• The flexibility to organize and manage media however I choose

Most importantly, I wanted the storage backend to be interchangeable.

Today it's Cloudflare R2.

Tomorrow it could be Blossom, S3, a local server, or something else entirely.

The editor shouldn't care where an image lives. It should simply upload the file and return a URL.

That's the philosophy behind Continuum.

Own your identity, your content and your media.

The provider is an implementation detail.
