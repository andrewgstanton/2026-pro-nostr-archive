7/3/2026 — Continuum Gets a Media Library

Today I added the first version of a Media Library to Continuum.

Instead of relying on external image hosts for article assets, Continuum can now upload images directly to my own media service backed by Cloudflare R2.

Each image receives a stable URL under my own domain, for example:

```https://media.mycontinuum.xyz/2026/pro/nostr/<npub>/images/...```

The Media Library currently supports:

- Uploading images

- Automatic safe filenames

- Duplicate filename handling

- Viewing uploaded images

- Copying image URLs

- Deleting images

- Synchronizing the media index while preserving metadata

Image metadata is stored separately in an index.json file, allowing tags and other descriptive information to evolve without changing the underlying storage layout.

This is only the first iteration, but it's already proving much more convenient for writing articles. The long-term goal is to support multiple storage backends—including Cloudflare R2, S3, Blossom, and others—behind a common interface so Continuum remains independent of any single media provider.
